\section{Relación con otros proyectos}

El sistema depende de la librería nativa epanet2\_64bit.dll, ya que usa esta librería como motor de cálculo. La librería cuenta con 54 funciones dentro de las cuales se encuentran funciones para correr las simulaciones, modificar y obtener las configuraciones de la red, modificar los elementos que conforman la red y generar reportes.

Las llamadas desde Java a la librería nativa serán realizadas a través de la librería \textit{Java Native Access} (JNA).

Adicionalmente, el sistema toma la arquitectura utilizada por el framework de optimización multiobjetivo Jmetal como base para agregar los algoritmos, operadores y problemas. Esta arquitectura será modificada según se necesite para satisfacer los requisitos del sistema.
